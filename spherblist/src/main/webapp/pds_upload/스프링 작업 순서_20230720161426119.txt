[스프링 작업 순서]

[1] get방식 - 화면 보여주는 페이지
	(입력화면, 수정화면, 삭제 화면 등)

1. 링크에 매핑 url 지정하기
2. 컨트롤러에서 메서드(핸들러) 틀 잡기
3. 뷰페이지 리턴

    DB작업을 하는 경우
3.  post방식과 동일

4. 뷰페이지에서 화면 출력하는 코드 작성

[2] post방식 - 실제적인 처리하는 페이지
	(입력처리, 수정처리, 삭제 처리 등)

1. Form 태그의 action 속성에 매핑 url 지정
2. 컨트롤러에서 메서드(핸들러) 틀 잡기
3. 컨트롤러에서 
	(1) 파라미터 읽어오기
	(2) db작업
		가) 매퍼 xml에서 쿼리문 작성
		나) DAO에서 매핑 구문에 해당하는
			메서드 선언부 만들기 (매핑구문 실행)
		다) Service 인터페이스에 메서드 선언부 만들기
		라) ServiceImpl (서비스 구현체)에 로직 작성
		마) 다시 컨트롤러로 돌아가서
			서비스의 메서드 호출
	(3) 모델에 결과 저장
	(4) 뷰페이지 리턴
		리다이렉트하거나 
		message.jsp로 포워드해서 메시지 보여주고 redirect되도록 한다

[3] get/post => 일반적인 조회의 경우
	(목록, 상세보기 등)

1. 링트나 Form태그의 action속성에 매핑 url 지정
2. 컨트롤러 틀 잡기
3. 컨트롤러에서 
	post방식과 동일
	단, (4) 뷰페이지 리턴 후
4. 뷰페이지 (jsp) 에서 화면 출력하는 코드작성
